SweetBot.design

Læring

Under dette forløb lærte jeg om projektplanlægning, informationsarkitektur og responsive design. Forløbet sluttede med et gruppeprojekt SweetBot.design, hvor min gruppe og jeg skulle implementere givne guidelines og designe en hjemmeside for det fiktive firma SweetBot og udvikling af e-commerce HotBot. Nedenfor kan man læse om vores GitHub strategi, responsive design og forbedringer, der er blevet foretaget. Den fulde rapport kan man finde nederst på siden.

GitHub

Strategien for GitHub struktur i dette projekt var, at committe til en udviklingsbranch, hvor vi derefter mergede med vores master. Under hele forløbet kodede jeg sammen med min gruppe, så der var plads til at lære af hinanden, reflektere og snakke højt om problemstillinger.
Se repository her

Responsive design

Hjemmesiden er responsive til desktop, tablet og mobil. I den sammenhæng blev der gjort brug af Simple Grid, udviklet af Dallas Bass, som vi navngav efter behov og tilføjede flere detaljer, så CSS-koden kunne benyttes til vores behov, nemlig; small(mobil), medium(tablet) og large(desktop).

Brugertest og forbedringer

Hjemmesiden blev både testet med tænk-højt-testen og gangstertesten. Disse gav en indikation på mangler og problemstillinger, som brugeren kan løbe ind i på hjemmesiden. Nedenfor kan man læse om resultaterne fra brugertestene, alle kan ses i den forbedrede udgave af hjemmesiden.
1. Testpersonerne manglede ’Book møde’ knap på siden ’Services’.
2. Placeringen af kompetencer skaber forvirring, når de er øverst på siden ’Om os’, derfor rykkes de nedenunder den enkelte medarbejder.
3. Derudover skal CEO’s tilføjes til siden ’Om os’, så alle medarbejdere kan ses på en side.
4. Tilføjelse af overskrifter på siderne i mobilformat, så brugeren ved hvor man befinder sig.
